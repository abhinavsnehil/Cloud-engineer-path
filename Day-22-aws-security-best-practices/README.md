## Shared Responsibility Model
- AWS secures the cloud infrastructure
- Customer secures data, OS, and configurations

## IAM Best Practices
- Follow least privilege principle
- Use IAM roles instead of access keys
- Enable MFA for critical accounts

## Network Security
- Use security groups as firewalls
- Restrict inbound and outbound traffic
- Use private subnets for sensitive resources

## Data Protection
- Encrypt data at rest and in transit
- Use KMS for key management
- Regularly backup data

## Monitoring and Compliance
- Enable CloudTrail for API activity
- Use CloudWatch for monitoring
- Review logs regularly

## Key Takeaways
- Security is a shared responsibility
- IAM and network controls are critical
- Monitoring ensures compliance and visibility
