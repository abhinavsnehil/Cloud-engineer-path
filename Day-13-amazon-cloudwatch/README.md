## What is CloudWatch
- Monitoring and observability service
- Collects metrics, logs, and events
- Helps detect operational issues

## Metrics
- Default metrics for AWS services
- Custom metrics supported
- Examples: CPUUtilization, DiskReadOps

## Logs
- Centralized log storage
- Used for application and system logs
- Log groups and log streams

## Alarms
- Trigger actions based on metric thresholds
- Can send notifications or scale resources

## Dashboards
- Visualize metrics using graphs
- Single-pane view of system health

## Events (EventBridge)
- Respond to changes in AWS resources
- Enables automation and event-driven architecture
